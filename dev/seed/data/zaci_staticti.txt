# =============================================================================
# Statičtí žáci vývojového prostředí BakaKeeperu
#
# Tito žáci existují vždy – nezávisí na --per-class ani na jmenných zásobnících.
# Jsou vhodní pro ruční sledování konkrétních scénářů v testech a debuggingu.
#
# Formát:  INTERN_KOD;JMENO;PRIJMENI;TRIDA;EVID_DO;E_MAIL
#
# INTERN_KOD  – unikátní kód; doporučujeme prefix ZFIX aby se nepřekrýval
#               s algoritmicky generovanými Z{05d}
# JMENO       – křestní jméno
# PRIJMENI    – příjmení
# TRIDA       – zkratka třídy (musí odpovídat záznamu v tridy.txt)
# EVID_DO     – datum ukončení evidence ve formátu RRRR-MM-DD;
#               prázdné = aktivní žák
# E_MAIL      – e-mail; prázdné = vygeneruje se automaticky jako
#               prijmeni.jmeno@<doména> (s dedupl. číslem pokud kolize)
#
# Statičtí žáci mají přednost při přiřazení základního (bečíslicového)
# e-mailu před algoritmicky generovanými žáky.
# Zákonní zástupci se pro statické žáky negenerují – přidejte ručně do
# statickych_zzd a statickych_zzr pokud je potřebujete.
#
# Řádky začínající # a prázdné řádky jsou ignorovány.
# =============================================================================
